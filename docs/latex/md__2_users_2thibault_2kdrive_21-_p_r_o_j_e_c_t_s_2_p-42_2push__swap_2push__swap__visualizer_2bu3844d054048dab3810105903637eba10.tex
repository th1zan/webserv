\label{_specifying-which-tests-to-run}%
 \hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu3844d054048dab3810105903637eba10_autotoc_md192}{}\doxysubsection{\texorpdfstring{Specifying which tests to run}{Specifying which tests to run}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu3844d054048dab3810105903637eba10_autotoc_md192}

\begin{DoxyPre}<test-\/spec> ...\end{DoxyPre}


Test cases, wildcarded test cases, tags and tag expressions are all passed directly as arguments. Tags are distinguished by being enclosed in square brackets.

If no test specs are supplied then all test cases, except "{}hidden"{} tests, are run. \doxylink{class_a}{A} test is hidden by giving it any tag starting with (or just) a period ({\ttfamily .}) -\/ or, in the deprecated case, tagged {\ttfamily \mbox{[}hide\mbox{]}} or given name starting with `'./\textquotesingle{}{\ttfamily . To specify hidden tests from the command line}{\ttfamily \mbox{[}.\mbox{]}}{\ttfamily or}{\ttfamily \mbox{[}hide\mbox{]}}\`{} can be used {\itshape regardless of how they were declared}.

Specs must be enclosed in quotes if they contain spaces. If they do not contain spaces the quotes are optional.

Wildcards consist of the {\ttfamily \texorpdfstring{$\ast$}{*}} character at the beginning and/or end of test case names and can substitute for any number of any characters (including none).

Test specs are case insensitive.

If a spec is prefixed with {\ttfamily exclude\+:} or the {\ttfamily \texorpdfstring{$\sim$}{\string~}} character then the pattern matches an exclusion. This means that tests matching the pattern are excluded from the set -\/ even if a prior inclusion spec included them. Subsequent inclusion specs will take precedence, however. Inclusions and exclusions are evaluated in left-\/to-\/right order.

Test case examples\+:


\begin{DoxyPre}thisTestOnly            Matches the test case called, 'thisTestOnly'
"{}this test only"{}        Matches the test case called, 'this test only'
these*                  Matches all cases starting with 'these'
exclude:notThis         Matches all tests except, 'notThis'
\string~notThis                Matches all tests except, 'notThis'
\string~*private*              Matches all tests except those that contain 'private'
a* \string~ab* abc             Matches all tests that start with 'a', except those that
                        start with 'ab', except 'abc', which is included
-\/\# [\#somefile]          Matches all tests from the file 'somefile.cpp'
\end{DoxyPre}


Names within square brackets are interpreted as tags. \doxylink{class_a}{A} series of tags form an AND expression whereas a comma-\/separated sequence forms an OR expression. e.\+g.\+:


\begin{DoxyPre}[one][two],[three]\end{DoxyPre}
 This matches all tests tagged {\ttfamily \mbox{[}one\mbox{]}} and {\ttfamily \mbox{[}two\mbox{]}}, as well as all tests tagged {\ttfamily \mbox{[}three\mbox{]}}

Test names containing special characters, such as {\ttfamily ,} or {\ttfamily \mbox{[}} can specify them on the command line using {\ttfamily \textbackslash{}}. {\ttfamily \textbackslash{}} also escapes itself.

\begin{DoxyPre}-\/r, -\/-\/reporter <reporter>\end{DoxyPre}


\doxylink{class_a}{A} reporter is an object that formats and structures the output of running tests, and potentially summarises the results. By default a console reporter is used that writes, IDE friendly, textual output. Catch comes bundled with some alternative reporters, but more can be added in client code.~\newline
 The bundled reporters are\+:


\begin{DoxyPre}-\/r console
-\/r compact
-\/r xml
-\/r junit
\end{DoxyPre}


The JUnit reporter is an xml format that follows the structure of the JUnit XML Report ANT task, as consumed by a number of third-\/party tools, including Continuous Integration servers such as Hudson. If not otherwise needed, the standard XML reporter is preferred as this is a streaming reporter, whereas the Junit reporter needs to hold all its results until the end so it can write the overall results into attributes of the root node.

\label{_aborting-after-a-certain-number-of-failures}%
 \hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu3844d054048dab3810105903637eba10_autotoc_md196}{}\doxysubsection{\texorpdfstring{Aborting after a certain number of failures}{Aborting after a certain number of failures}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu3844d054048dab3810105903637eba10_autotoc_md196}

\begin{DoxyPre}-\/a, -\/-\/abort
-\/x, -\/-\/abortx [<failure threshold>]
\end{DoxyPre}


If a {\ttfamily REQUIRE} assertion fails the test case aborts, but subsequent test cases are still run. If a {\ttfamily CHECK} assertion fails even the current test case is not aborted.

Sometimes this results in a flood of failure messages and you\textquotesingle{}d rather just see the first few. Specifying {\ttfamily -\/a} or {\ttfamily -\/-\/abort} on its own will abort the whole test run on the first failed assertion of any kind. Use {\ttfamily -\/x} or {\ttfamily -\/-\/abortx} followed by a number to abort after that number of assertion failures.

 \hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu3844d054048dab3810105903637eba10_autotoc_md200}{}\doxysubsection{\texorpdfstring{Eliding assertions expected to throw}{Eliding assertions expected to throw}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu3844d054048dab3810105903637eba10_autotoc_md200}

\begin{DoxyPre}-\/e, -\/-\/nothrow\end{DoxyPre}


Skips all assertions that test that an exception is thrown, e.\+g. {\ttfamily REQUIRE\+\_\+\+THROWS}.

These can be a nuisance in certain debugging environments that may break when exceptions are thrown (while this is usually optional for handled exceptions, it can be useful to have enabled if you are trying to track down something unexpected).

Sometimes exceptions are expected outside of one of the assertions that tests for them (perhaps thrown and caught within the code-\/under-\/test). The whole test case can be skipped when using {\ttfamily -\/e} by marking it with the {\ttfamily \mbox{[}!throws\mbox{]}} tag.

When running with this option any throw checking assertions are skipped so as not to contribute additional noise. Be careful if this affects the behaviour of subsequent tests.

When set to {\ttfamily yes} Catch will report the duration of each test case, in milliseconds. Note that it does this regardless of whether a test case passes or fails. Note, also, the certain reporters (e.\+g. Junit) always report test case durations regardless of this option being set or not.

\begin{DoxyPre}-\/-\/rng-\/seed <'time'|number>\end{DoxyPre}


Sets a seed for the random number generator using {\ttfamily std\+::srand()}. If a number is provided this is used directly as the seed so the random pattern is repeatable. Alternatively if the keyword {\ttfamily time} is provided then the result of calling {\ttfamily std\+::time(0)} is used and so the pattern becomes unpredictable. In some cases, you might need to pass the keyword {\ttfamily time} in double quotes instead of single quotes.

In either case the actual value for the seed is printed as part of Catch\textquotesingle{}s output so if an issue is discovered that is sensitive to test ordering the ordering can be reproduced -\/ even if it was originally seeded from {\ttfamily std\+::time(0)}.

\begin{DoxyPre}-\/c, -\/-\/section <section name>\end{DoxyPre}


To limit execution to a specific section within a test case, use this option one or more times. To narrow to sub-\/sections use multiple instances, where each subsequent instance specifies a deeper nesting level.

E.\+g. if you have\+:


\begin{DoxyPre}
TEST\_CASE( "{}Test"{} ) \{
  SECTION( "{}sa"{} ) \{
    SECTION( "{}sb"{} ) \{
      /*...*/
    \}
    SECTION( "{}sc"{} ) \{
      /*...*/
    \}
  \}
  SECTION( "{}sd"{} ) \{
    /*...*/
  \}
\}
\end{DoxyPre}


Then you can run {\ttfamily sb} with\+: 
\begin{DoxyPre}./MyExe Test -\/c sa -\/c sb\end{DoxyPre}


Or run just {\ttfamily sd} with\+: 
\begin{DoxyPre}./MyExe Test -\/c sd\end{DoxyPre}


To run all of {\ttfamily sa}, including {\ttfamily sb} and {\ttfamily sc} use\+: 
\begin{DoxyPre}./MyExe Test -\/c sa\end{DoxyPre}


There are some limitations of this feature to be aware of\+:
\begin{DoxyItemize}
\item Code outside of sections being skipped will still be executed -\/ e.\+g. any set-\/up code in the TEST\+\_\+\+CASE before the start of the first section.
\end{DoxyItemize}
\begin{DoxyItemize}
\item At time of writing, wildcards are not supported in section names.
\item If you specify a section without narrowing to a test case first then all test cases will be executed (but only matching sections within them).
\end{DoxyItemize}

\begin{DoxyPre}-\/\#, -\/-\/filenames-\/as-\/tags\end{DoxyPre}


When this option is used then every test is given an additional tag which is formed of the unqualified filename it is found in, with any extension stripped, prefixed with the {\ttfamily \#} character.

\begin{DoxyPre}-\/-\/use-\/colour <yes|no|auto>\end{DoxyPre}


Catch colours output for terminals, but omits colouring when it detects that output is being sent to a pipe. This is done to avoid interfering with automated processing of output.

{\ttfamily -\/-\/use-\/colour yes} forces coloured output, {\ttfamily -\/-\/use-\/colour no} disables coloured output. The default behaviour is {\ttfamily -\/-\/use-\/colour auto}.
