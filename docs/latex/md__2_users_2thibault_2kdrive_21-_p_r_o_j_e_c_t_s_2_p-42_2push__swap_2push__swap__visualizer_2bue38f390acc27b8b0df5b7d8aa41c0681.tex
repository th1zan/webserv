The code in imgui.\+cpp embeds a copy of \textquotesingle{}Proggy\+Clean.\+ttf\textquotesingle{} (by Tristan Grimmer), a 13 pixels high, pixel-\/perfect font used by default. We embed it in the source code so you can use Dear Im\+Gui without any file system access. Proggy\+Clean does not scale smoothly, therefore it is recommended that you load your own file when using Dear Im\+Gui in an application aiming to look nice and wanting to support multiple resolutions.

You may also load external .TTF/.OTF files. In the \href{https://github.com/ocornut/imgui/tree/master/misc/fonts}{\texttt{ misc/fonts/}} folder you can find a few suggested fonts, provided as a convenience.

{\bfseries{Font Atlas too large?}}


\begin{DoxyItemize}
\item If you have very large number of glyphs or multiple fonts, the texture may become too big for your graphics API. The typical result of failing to upload a texture is if every glyphs appears as white rectangles.
\item Mind the fact that some graphics drivers have texture size limitation. If you are building a PC application, mind the fact that your users may use hardware with lower limitations than yours.
\end{DoxyItemize}

Some solutions\+:


\begin{DoxyEnumerate}
\item Reduce glyphs ranges by calculating them from source localization data. You can use the {\ttfamily \doxylink{struct_im_font_glyph_ranges_builder}{Im\+Font\+Glyph\+Ranges\+Builder}} for this purpose and rebuilding your atlas between frames when new characters are needed. This will be the biggest win!
\item You may reduce oversampling, e.\+g. {\ttfamily font\+\_\+config.\+OversampleH = 2}, this will largely reduce your texture size. Note that while OversampleH = 2 looks visibly very close to 3 in most situations, with OversampleH = 1 the quality drop will be noticeable.
\item Set {\ttfamily io.\+Fonts.\+Tex\+Desired\+Width} to specify a texture width to minimize texture height (see comment in {\ttfamily Im\+Font\+Atlas\+::\+Build()} function).
\item Set {\ttfamily io.\+Fonts.\+Flags \texorpdfstring{$\vert$}{|}= Im\+Font\+Atlas\+Flags\+\_\+\+No\+Power\+Of\+Two\+Height;} to disable rounding the texture height to the next power of two.
\item Read about oversampling \href{https://github.com/nothings/stb/blob/master/tests/oversample}{\texttt{ here}}.
\item To support the extended range of unicode beyond 0x\+FFFF (e.\+g. emoticons, dingbats, symbols, shapes, ancient languages, etc...) add {\ttfamily \#define IMGUI\+\_\+\+USE\+\_\+\+WCHAR32}in your {\ttfamily \doxylink{imconfig_8h_source}{imconfig.\+h}}.
\end{DoxyEnumerate}\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bue38f390acc27b8b0df5b7d8aa41c0681_autotoc_md759}{}\doxysubsubsubsubsection{\texorpdfstring{\protect\doxysectlink{index}{Return to Index}{0}}{{Return to Index}}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bue38f390acc27b8b0df5b7d8aa41c0681_autotoc_md759}

\begin{DoxyItemize}
\item Dear Im\+Gui uses \doxylink{imstb__truetype_8h_source}{imstb\+\_\+truetype.\+h} to rasterize fonts (with optional oversampling). This technique and its implementation are not ideal for fonts rendered at small sizes, which may appear a little blurry or hard to read.

\item Free\+Type supports auto-\/hinting which tends to improve the readability of small fonts.
\item Read documentation in the \href{https://github.com/ocornut/imgui/tree/master/misc/freetype}{\texttt{ misc/freetype/}} folder.
\item Correct s\+RGB space blending will have an important effect on your font rendering quality.
