This page documents current deprecations and upcoming planned changes inside Catch2. The difference between these is that a deprecated feature will be removed, while a planned change to a feature means that the feature will behave differently, but will still be present. Obviously, either of these is a breaking change, and thus will not happen until at least the next major release.\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md260}{}\doxysubsection{\texorpdfstring{Deprecations}{Deprecations}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md260}
\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md261}{}\doxysubsubsection{\texorpdfstring{{\ttfamily -\/-\/list-\/\texorpdfstring{$\ast$}{*}} return values}{{\ttfamily -\/-\/list-\/\texorpdfstring{$\ast$}{*}} return values}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md261}
The return codes of the {\ttfamily -\/-\/list-\/\texorpdfstring{$\ast$}{*}} family of command line arguments will no longer be equal to the number of tests/tags/etc found, instead it will be 0 for success and non-\/zero for failure.\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md262}{}\doxysubsubsection{\texorpdfstring{{\ttfamily -\/-\/list-\/test-\/names-\/only}}{{\ttfamily -\/-\/list-\/test-\/names-\/only}}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md262}
{\ttfamily -\/-\/list-\/test-\/names-\/only} command line argument will be removed.\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md263}{}\doxysubsubsection{\texorpdfstring{{\ttfamily ANON\+\_\+\+TEST\+\_\+\+CASE}}{{\ttfamily ANON\+\_\+\+TEST\+\_\+\+CASE}}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md263}
{\ttfamily ANON\+\_\+\+TEST\+\_\+\+CASE} is scheduled for removal, as it can be fully replaced by a {\ttfamily TEST\+\_\+\+CASE} with no arguments.\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md264}{}\doxysubsubsection{\texorpdfstring{Secondary description amongst tags}{Secondary description amongst tags}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md264}
Currently, the tags part of {\ttfamily TEST\+\_\+\+CASE} (and others) macro can also contain text that is not part of tags. This text is then separated into a "{}description"{} of the test case, but the description is then never used apart from writing it out for {\ttfamily -\/-\/list-\/tests -\/v high}.

Because it isn\textquotesingle{}t actually used nor documented, and brings complications to Catch2\textquotesingle{}s internals, description support will be removed.\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md265}{}\doxysubsubsection{\texorpdfstring{Source\+Line\+Info\+::empty()}{Source\+Line\+Info\+::empty()}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md265}
There should be no reason to ever have an empty {\ttfamily Source\+Line\+Info}, so the method will be removed.\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md266}{}\doxysubsubsection{\texorpdfstring{Composing lvalues of already composed matchers}{Composing lvalues of already composed matchers}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md266}
Because a significant bug in this use case has persisted for 2+ years without a bug report, and to simplify the implementation, code that composes lvalues of composed matchers will not compile. That is, this code will no longer work\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{auto}\ m1\ =\ Contains(\textcolor{stringliteral}{"{}string"{}});}
\DoxyCodeLine{\textcolor{keyword}{auto}\ m2\ =\ Contains(\textcolor{stringliteral}{"{}random"{}});}
\DoxyCodeLine{\textcolor{keyword}{auto}\ composed1\ =\ m1\ ||\ m2;}
\DoxyCodeLine{\textcolor{keyword}{auto}\ m3\ =\ Contains(\textcolor{stringliteral}{"{}different"{}});}
\DoxyCodeLine{\textcolor{keyword}{auto}\ composed2\ =\ composed1\ ||\ m3;}
\DoxyCodeLine{REQUIRE\_THAT(\mbox{\hyperlink{structfoo}{foo}}(),\ !composed1);}
\DoxyCodeLine{REQUIRE\_THAT(\mbox{\hyperlink{structfoo}{foo}}(),\ composed2);}

\end{DoxyCode}


Instead you will have to write this\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{auto}\ m1\ =\ Contains(\textcolor{stringliteral}{"{}string"{}});}
\DoxyCodeLine{\textcolor{keyword}{auto}\ m2\ =\ Contains(\textcolor{stringliteral}{"{}random"{}});}
\DoxyCodeLine{\textcolor{keyword}{auto}\ m3\ =\ Contains(\textcolor{stringliteral}{"{}different"{}});}
\DoxyCodeLine{REQUIRE\_THAT(\mbox{\hyperlink{structfoo}{foo}}(),\ !(m1\ ||\ m2));}
\DoxyCodeLine{REQUIRE\_THAT(\mbox{\hyperlink{structfoo}{foo}}(),\ m1\ ||\ m2\ ||\ m3);}

\end{DoxyCode}
\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md267}{}\doxysubsubsection{\texorpdfstring{{\ttfamily Parse\+And\+Add\+Catch\+Tests.\+cmake}}{{\ttfamily Parse\+And\+Add\+Catch\+Tests.\+cmake}}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md267}
The CMake/\+CTest integration using {\ttfamily Parse\+And\+Add\+Catch\+Tests.\+cmake} is deprecated, as it can be replaced by {\ttfamily Catch.\+cmake} that provides the function {\ttfamily catch\+\_\+discover\+\_\+tests} to get tests directly from a CMake target via the command line interface instead of parsing C++ code with regular expressions.\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md268}{}\doxysubsection{\texorpdfstring{Planned changes}{Planned changes}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md268}
\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md269}{}\doxysubsubsection{\texorpdfstring{Reporter verbosities}{Reporter verbosities}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md269}
The current implementation of verbosities, where the reporter is checked up-\/front whether it supports the requested verbosity, is fundamentally misguided and will be changed. The new implementation will no longer check whether the specified reporter supports the requested verbosity, instead it will be up to the reporters to deal with verbosities as they see fit (with an expectation that unsupported verbosities will be, at most, warnings, but not errors).\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md270}{}\doxysubsubsection{\texorpdfstring{Output format of {\ttfamily -\/-\/list-\/\texorpdfstring{$\ast$}{*}} command line parameters}{Output format of {\ttfamily -\/-\/list-\/\texorpdfstring{$\ast$}{*}} command line parameters}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md270}
The various list operations will be piped through reporters. This means that e.\+g. XML reporter will write the output as machine-\/parseable XML, while the Console reporter will keep the current, human-\/oriented output.\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md271}{}\doxysubsubsection{\texorpdfstring{{\ttfamily CHECKED\+\_\+\+IF} and {\ttfamily CHECKED\+\_\+\+ELSE}}{{\ttfamily CHECKED\+\_\+\+IF} and {\ttfamily CHECKED\+\_\+\+ELSE}}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md271}
To make the {\ttfamily CHECKED\+\_\+\+IF} and {\ttfamily CHECKED\+\_\+\+ELSE} macros more useful, they will be marked as "{}\+OK to fail"{} ({\ttfamily Catch\+::\+Result\+Disposition\+::\+Suppress\+Fail} flag will be added), which means that their failure will not fail the test, making the {\ttfamily else} actually useful.\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md272}{}\doxysubsubsection{\texorpdfstring{Change semantics of {\ttfamily \mbox{[}.\mbox{]}} and tag exclusion}{Change semantics of {\ttfamily \mbox{[}.\mbox{]}} and tag exclusion}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md272}
Currently, given these 2 tests 
\begin{DoxyCode}{0}
\DoxyCodeLine{TEST\_CASE(\textcolor{stringliteral}{"{}A"{}},\ \textcolor{stringliteral}{"{}[.][foo]"{}})\ \{\}}
\DoxyCodeLine{TEST\_CASE(\textcolor{stringliteral}{"{}B"{}},\ \textcolor{stringliteral}{"{}[.][bar]"{}})\ \{\}}

\end{DoxyCode}
 specifying {\ttfamily \mbox{[}foo\mbox{]}} as the testspec will run test "{}\+A"{} and specifying {\ttfamily \texorpdfstring{$\sim$}{\string~}\mbox{[}foo\mbox{]}} will run test "{}\+B"{}, even though it is hidden. Also, specifying {\ttfamily \texorpdfstring{$\sim$}{\string~}\mbox{[}baz\mbox{]}} will run both tests. This behaviour is often surprising and will be changed so that hidden tests are included in a run only if they positively match a testspec.\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md273}{}\doxysubsubsection{\texorpdfstring{Console Colour API}{Console Colour API}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md273}
The API for Catch2\textquotesingle{}s console colour will be changed to take an extra argument, the stream to which the colour code should be applied.\hypertarget{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md274}{}\doxysubsubsection{\texorpdfstring{Type erasure in the {\ttfamily Predicate\+Matcher}}{Type erasure in the {\ttfamily Predicate\+Matcher}}}\label{md__2_users_2thibault_2kdrive_21-_p_r_o_j_e_c_t_s_2_p-42_2push__swap_2push__swap__visualizer_2bu65b1e4d1558c172dd086b1e4a75c51eb_autotoc_md274}
Currently, the {\ttfamily Predicate\+Matcher} uses {\ttfamily std\+::function} for type erasure, so that type of the matcher is always {\ttfamily Predicate\+Matcher\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}}, regardless of the type of the predicate. Because of the high compilation overhead of {\ttfamily std\+::function}, and the fact that the type erasure is used only rarely, {\ttfamily Predicate\+Matcher} will no longer be type erased in the future. Instead, the predicate type will be made part of the Predicate\+Matcher\textquotesingle{}s type.
