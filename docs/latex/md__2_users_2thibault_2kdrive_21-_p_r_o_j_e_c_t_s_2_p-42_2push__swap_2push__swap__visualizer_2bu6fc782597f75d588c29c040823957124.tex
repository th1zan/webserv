To run on a local machine\+:
\begin{DoxyItemize}
\item {\ttfamily make serve} will use Python3 to spawn a local webserver, you can then browse \href{http://localhost:8000}{\texttt{ http\+://localhost\+:8000}} to access your build.
\item Otherwise, generally you will need a local webserver\+:
\begin{DoxyItemize}
\item Quoting \href{https://emscripten.org/docs/getting_started/Tutorial.html\#generating-html}{\texttt{ https\+://emscripten.\+org/docs/getting\+\_\+started}}\+:~\newline
 {\itshape "{}\+Unfortunately several browsers (including Chrome, Safari, and Internet Explorer) do not support file\+:// \mbox{[}\+XHR\mbox{]}(https\+://emscripten.\+org/docs/site/glossary.\+html\#term-\/xhr) requests, and can’t load extra files needed by the HTML (like a .\+wasm file, or packaged file data as mentioned lower down). For these browsers you’ll need to serve the files using a \mbox{[}local webserver\mbox{]}(https\+://emscripten.\+org/docs/getting\+\_\+started/\+FAQ.\+html\#faq-\/local-\/webserver) and then open http\+://localhost\+:8000/hello.\+html."{}}
\item Emscripten SDK has a handy {\ttfamily emrun} command\+: {\ttfamily emrun web/example\+\_\+emscripten\+\_\+opengl3.\+html -\/-\/browser firefox} which will spawn a temporary local webserver (in Firefox). See \href{https://emscripten.org/docs/compiling/Running-html-files-with-emrun.html}{\texttt{ https\+://emscripten.\+org/docs/compiling/\+Running-\/html-\/files-\/with-\/emrun.\+html}} for details.
\item You may use Python 3 builtin webserver\+: {\ttfamily python -\/m http.\+server -\/d web} (this is what {\ttfamily make serve} uses).
\item You may use Python 2 builtin webserver\+: {\ttfamily cd web \&\& python -\/m Simple\+HTTPServer}.
\end{DoxyItemize}
