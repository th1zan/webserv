\doxysubsection{Detailed Description}
Interface of the delegate of the window implementation. 

We don\textquotesingle{}t create an interface here because Obj-\/C doesn\textquotesingle{}t allow multiple inheritance (\doxylink{interface_s_f_view_controller}{SFView\+Controller} and \doxylink{interface_s_f_window_controller}{SFWindow\+Controller} don\textquotesingle{}t have the same parent classes). Unfortunately this means we have to duplicate some code.

Everything is done via a class that implement this protocol. There are two of these classes\+:

\doxylink{interface_s_f_view_controller}{SFView\+Controller} and \doxylink{interface_s_f_window_controller}{SFWindow\+Controller}

The requester is a Window\+Impl\+Cocoa. It\textquotesingle{}s used to send back event via these functions\+:

window\+Closed, window\+Resized, window\+Lost\+Focus, window\+Gained\+Focus

mouse\+Down\+At, mouse\+Up\+At, mouse\+Moved\+At, mouse\+Wheel\+Scrolled\+At, mouse\+Moved\+In, mouse\+Moved\+Out

key\+Down, key\+Up, text\+Entered

Note\+: Joysticks are not bound to a view or window thus they\textquotesingle{}re not managed by a class implementing this protocol.
